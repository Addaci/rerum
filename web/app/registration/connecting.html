Connecting to RERUM

To limit the possibility of abuse without putting too many obstacles up, writing to the annotation store is restricted to servers (no CORS) who have registered their IP (no ranges or named referrers). This is not permanent, but the full plans for RERUM can be found elsewhere on the site.

The current public iteration of the RERUM annotation store is hosted on a virtual server in the development area of OngCDH at Saint Louis University. The backend is written in Java and MongoDB, with all requests currently over HTTP.

All read requests (GET) are open by default. To write to RERUM, Follow these simple steps:

  1. Ensure your requests are all coming from a single server or proxy and register the IP address. here in RERUM.
  2. Follow the API (API page) to create a new object or update an existing one and you will receive the location of the new object in the response. Your registered information is stored with all successful writes to the annotation store.
