Version 0

Though in the early stages, we provide several endpoints for creating, updating, or reading any object in the database from any registered server. All requests are HTTP POST. Objects are known by the assigned @id or objectID assigned by MongoDB.

This version of RERUM runs on Ubuntu/Tomcat/Java/MongoDB. Much of the code refers to "annotation" objects, but any valid JSON object can be placed into RERUM. Strictly speaking, an "annotation" is the content:{} wrapper around the data.

Actions

Get an entire namespace

Retrieve all namespaced objects from the store including the latest version, all forks and revisions.

anno/getAllMyAnnotations.action

Query with stringified "content":{"namespace":YOUR_NAMESPACE}

Returns a List of objects.

Get all versions of an object.

Retrieve all versions of a single object.

anno/getAllVersionsOfAnnotationByObjectID.action

Query with stringified "content":{"objectID":ID}

Returns a List of objects.

Query for object by properties.

Retrieve all objects that match the property or properties values.

anno/getAnnotationByProperties.action

Query with stringified "content":{KEY:VALUE}

Returns a List of objects.

Create an object.

Place a new object in RERUM.

anno/saveNewAnnotation.action

"content":{NEW_OBJECT} Include complete object.

Returns HTTP status and object @id.

Update an object.

Replace the object in RERUM with the updated version.

anno/updateAnnotation.action

"content":{"objectID":ID} Include complete object, even unchanged properties.

Returns HTTP status.

Revise an object.

Create a newer version of the object in RERUM with the updated version.

anno/saveNewVersionOfAnnotation.action

"content":{"objectID":ID} Include complete new version, even unchanged properties.

Returns HTTP status.

Delete an object.

Remove an object from RERUM completely. Identify by objectID or @id.

anno/deleteAnnotationByObjectID.action anno/deleteAnnotationByAtID.action

"content":{"objectID":ID} or "content":{"@id":ID}.

Returns HTTP status.
